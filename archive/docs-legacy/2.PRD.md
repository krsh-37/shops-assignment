Assumption: the shared memory layer may use Mem0 or a compatible local store with the same read and write contract.

## 1. User Personas
1. Founder or operator who submits one brand idea and wants a complete launch package.
2. Internal operator who runs the workflow, answers clarification questions, and reviews progress.
3. Reviewer who checks that the output is complete, consistent, and ready to hand off.

## 2. User Goals and Pain Points
The founder wants a single conversation to produce a usable brand launch plan without coordinating multiple tools. The main pain points are slow back-and-forth, inconsistent outputs across teams, and repeated questions for information already provided.

The operator wants clear workflow status, predictable pauses for clarification, and outputs that are easy to validate. The main pain points are hidden failures, state loss between steps, and deliverables that do not line up across research, brand, store, ads, and SEO.

## 3. User Stories with Acceptance Criteria
1. As a founder, I want to submit one brand idea and receive a guided launch workflow. Given a valid idea, when I submit it, then the system starts one run and stores the idea in shared memory. Given a blank idea, when I submit it, then the system rejects the run.
2. As a founder, I want the system to ask key questions before expensive work starts. Given missing launch inputs, when the run begins, then the system asks batched clarification questions. Given unanswered questions, when the run is paused, then no downstream agent starts.
3. As an operator, I want to see progress across agents. Given a run in progress, when an agent completes, then the system shows the updated status. Given an agent fails, when the workflow continues, then the failure is visible without hiding prior completed work.
4. As a reviewer, I want a final launch bible that reflects all completed work. Given all required agent outputs exist, when the workflow ends, then the system produces a markdown bible and a structured JSON record. Given one required output is missing, when the report is generated, then the bible is marked incomplete.

## 4. Functional Requirements
1. **PRD-001** The system shall accept one founder prompt and create one run-scoped launch record. If the prompt is empty or malformed, the run shall not start.
2. **PRD-002** The orchestrator shall collect clarification questions before it starts any non-orchestrator agent. For this phase, non-orchestrator agents are Research, Visual, Domain, India GTM, Shopify, Performance Ads, SEO/GEO, and Launch Report. If answers are missing, the workflow shall pause.
3. **PRD-003** The shared memory store shall persist run-scoped inputs and agent outputs using the approved memory schema. If the store is unavailable, the run shall fail before agent execution.
4. **PRD-004** The workflow engine shall run Research and Domain in parallel. If one step fails to start, the other step shall still proceed.
5. **PRD-014** The workflow engine shall block Visual until Research finishes. If Research is incomplete, Visual shall remain pending.
6. **PRD-015** The workflow engine shall block Shopify until Visual and GTM finish. If either prerequisite is missing, Shopify shall remain pending.
7. **PRD-005** The Research Agent shall write one structured research report containing competitors, market size, whitespace, keywords, and India insight. If the founder idea is missing, the agent shall return a failure status.
8. **PRD-006** The Visual Agent shall write exactly three logo concept directions derived from prior research. If research data is missing, the agent shall not generate concepts.
9. **PRD-007** The Domain Agent shall write a ranked shortlist of domain candidates with a recommended name. If availability checks fail, the agent shall mark the affected candidates unavailable.
10. **PRD-008** The India GTM Agent shall write one India market launch plan with city priority, channel split, reel ideas, influencer brief, and week-one checklist. If research, visual, or domain context is missing, the agent shall stop.
11. **PRD-009** The Shopify Agent shall write one store asset package that uses the visual palette from `visual.palette` and the research keywords from `research.keywords`. If either source is missing, the agent shall fail.
12. **PRD-010** The Performance Ads Agent shall write one ad strategy package using `idea`, `research`, `visual`, and `domains` from shared memory for brand context. If any of those memory keys are missing, the agent shall fail without asking for the brand name again.
13. **PRD-011** The SEO/GEO Agent shall write one search plan that uses `research.keywords` and includes classic SEO content and GEO-ready FAQ content. If keyword data is missing, the agent shall fail.
14. **PRD-012** The Launch Report Agent shall write one final Brand Launch Bible in markdown and structured JSON from the outputs of Research, Visual, Domain, GTM, Shopify, Ads, and SEO/GEO. If any prerequisite output is missing, the report shall be marked incomplete.
15. **PRD-013** The delivery package shall include a runnable TypeScript and Mastra codebase. If the codebase cannot run locally, the delivery shall be incomplete.
16. **PRD-016** The delivery package shall include typed tools. If any required tool contract is missing, the delivery shall be incomplete.
17. **PRD-017** The delivery package shall include a README. If the README is missing, the delivery shall be incomplete.
18. **PRD-018** The delivery package shall include a design document. If the design document is missing, the delivery shall be incomplete.
19. **PRD-019** The delivery package shall include a demo artifact. If the demo artifact is missing, the delivery shall be incomplete.
20. **PRD-020** The delivery package shall include stubbed external integrations. If any required external interface is not stubbed or mocked, the delivery shall be incomplete.
21. **PRD-021** The orchestrator shall publish workflow status after each agent completes. If an agent fails, the failure shall remain visible without erasing prior completed work.

## 5. Non-Functional Requirements
1. **PRD-NFR-001** A complete local demo run shall finish in under 10 minutes from prompt submission to final bible output. If execution exceeds 10 minutes, the run shall be marked as degraded.
2. **PRD-NFR-002** Shared memory reads and writes shall remain isolated per run. If data from a prior run appears in the current run, the run shall fail validation.
