# Phase 01 PRD

Approved scope boundary: `PRD-001`.

## 1. Problem Summary
The system needs a safe entry point that accepts one founder prompt and creates one run-scoped launch record. This phase establishes the minimal run start behavior used by all later phases.

## 2. In-Scope Items
- Accept one founder prompt.
- Validate that the prompt is a non-empty string.
- Create one run-scoped launch record.
- Persist the record in the selected store.

## 3. Out-of-Scope Items
- Clarification questions.
- Shared memory schema beyond the run record.
- Agent execution.
- Workflow orchestration.

## 4. Assumptions
1. The run record can be stored in an in-memory implementation for this phase.
2. Node 24 can execute the TypeScript slice directly.
3. The run ID is generated by the implementation.

## 5. Constraints and Risks
- Invalid input must fail before any record is persisted.
- The phase must remain independent of later agent logic.

## 6. Open Questions
- None.

## 7. Success Criteria
- A valid founder prompt creates exactly one record.
- Empty or malformed input creates no record.
- Tests cover the valid and invalid path.
