00 What is OpenClaw
One idea in.
A live brand out.

A founder says: "I want to sell custom socks delivered in 10 minutes." OpenClaw takes it from there. Eight specialized agents run in a coordinated workflow — each one reading from shared memory, each one writing back what it learns.
The Core Thesis

"Each agent's output makes the next one smarter. By the time the Shopify agent runs, it already knows the brand name, the visual direction, the India marketing angle, and the SEO keywords — because the agents before it wrote all of that into shared memory."

01 The Eight Agents
Eight agents.
All required.

Each agent has a defined scope, a set of tools, a clarification protocol, and a Mem0 write contract. Build all eight in Mastra. Stubs are fine for external APIs — agent design is what we're scoring.
Agent 00 Orchestrator
Orchestrator
Routes the workflow · delegates to agents · maintains conversation

The user talks only to the Orchestrator. It parses intent, routes to the right agent, collects clarification answers, and sequences the workflow. It must handle partial completions (e.g. Research done but Visual still running) and present a live status panel to the user.
// mastra/agents/orchestrator.ts import { Agent } from '@mastra/core'; export const orchestrator = new Agent({ name: 'OpenClaw Orchestrator', instructions: `You are the brain of OpenClaw. You receive a brand idea and coordinate 7 specialist agents. Always read from Mem0 before delegating. Always ask clarifying questions BEFORE starting expensive agents. Present a status update after each agent completes.`, model: { provider: 'ANTHROPIC', name: 'claude-sonnet-4-5' }, tools: { delegateToAgent, readMem0, writeMem0, askUser }, });
Agent 01 Required
Research Agent
Market intel · competitor mapping · TAM sizing

Takes the brand idea. Searches for competitors in India and globally. Estimates market size. Identifies whitespace. Outputs a structured BrandResearchReport to Mem0 and a human-readable markdown summary to the user.
// Tools this agent uses: webSearch(query: string) // Tavily or Exa fetchPage(url: string) // scrape a URL writeMem0(key: 'research', value: BrandResearchReport) // Writes: competitors, market_size, whitespace, keywords
Clarification questions asked upfront

    What cities in India should we prioritise first?
    What's the price point you're targeting?
    Direct-to-consumer only, or open to quick-commerce channels?

Agent 02 Required
Visual Agent
Logo concepts · brand palette · Nanobanana 2

Reads Research report from Mem0. Generates 3 distinct logo concept directions using Nanobanana 2. Each concept gets a name, mood, and Nanobanana prompt. Outputs image URLs + brand palette to Mem0.
// Nanobanana 2 integration const generateLogoConcept = createTool({ id: 'nanobanana_generate', inputSchema: z.object({ prompt: z.string(), style: z.enum(['minimal','bold','playful']), aspect_ratio: z.string().default('1:1'), brand_context: z.string(), }), execute: async({ context }) => nanobanannaV2(context) });
Clarification questions asked upfront

    Serious and premium, or fun and quirky?
    Any colours the brand must have or must avoid?
    References you admire? (brands, not just logos)

Agent 03 Required
Domain Agent
Name ideation · availability check · ranked shortlist

Generates 15 brand name + domain combinations. Checks availability via RDAP or a DNS stub. Ranks them by memorability, .in/.com availability, and brand fit. Outputs top 5 to Mem0 with reasoning.
// Tools this agent uses: checkDomainAvailability( domain: string // "sockzy.in" ) → { available: boolean, price: number } writeMem0(key: 'domains', value: { top5: DomainResult[], recommended: string } )
Clarification questions asked upfront

    Hindi/vernacular name okay, or English only?
    Should the domain name describe the product or be abstract?

Agent 04 Required
India GTM Agent
India-first go-to-market · WhatsApp · Reels · D2C channels

Reads research + domain + visual from Mem0. Creates an India-specific GTM plan: city priority order, WhatsApp broadcast strategy, Instagram Reel content calendar (10 ideas), influencer tier (nano/micro for India), and channel budget split.
// Output written to Mem0: { launch_cities: ['Mumbai','Bengaluru','Pune'], channels: { instagram:'40%', whatsapp:'30%', google:'30%' }, reel_ideas: string[], // 10 hooks influencer_brief: string, week1_checklist: string[] }
Clarification questions asked upfront

    Launch budget for first 90 days (rough range)?
    Targeting corporates, students, or general consumers?
    Quick-commerce partnerships (Zepto, Blinkit) a priority?

Agent 05 Required
Shopify Agent
Store build · free theme · product catalog · homepage copy

Reads brand name, visual direction, logo URLs, and pricing from Mem0. Generates: a complete theme.liquid customisation patch for Dawn theme, 3 product listings with descriptions, homepage hero copy, and a collection setup JSON. Does not need a real Shopify store — outputs files a developer can apply.
// Output: a /shopify directory // ├── theme-settings.json ← colors, fonts from Mem0 brand // ├── products.json ← 3 products with copy // ├── homepage-sections.json← hero, value props, social proof // └── collections.json ← initial taxonomy generateShopifyAssets( brand: readMem0('brand'), visual: readMem0('visual'), gtm: readMem0('gtm') ) → ShopifyPackage
Clarification questions asked upfront

    How many SKUs to launch with (3–5 recommended)?
    COD needed, or card/UPI only?
    Any existing photos, or should copy assume generated images?

Agent 06 Required
Performance Ads Agent
Meta · Google · India-specific targeting · ad copy

Reads research, GTM, and visual from Mem0. Produces: 3 Meta ad creative briefs (with hooks, body, CTA), 2 Google Search campaign structures with keywords and match types, audience targeting spec for India, and a 30-day budget pacing plan starting at ₹500/day.
// Output written to Mem0: { meta_ads: [{ format:'Reel', hook:string, body:string, cta:string, audience: IndiaAudienceSpec, budget_day:number }], google_campaigns: GoogleCampaignSpec[], 30_day_plan: PacingPlan }
Clarification questions asked upfront

    Performance goal: awareness, trials, or repeat orders?
    Do you have video assets, or start with static?

Agent 07 Required
SEO / GEO Agent
Search + generative engine optimisation · India D2C

Does both: classic D2C SEO (keyword clusters, PDP schema, blog topics) AND GEO — optimising for ChatGPT, Perplexity, and Google AI Overviews. Generates 5 "answer-ready" FAQ pages the brand should publish to get cited by AI search. Writes all metadata to Mem0.
// GEO = Generative Engine Optimisation // Goal: appear in AI search answers for queries like // "best custom socks brand India" or "10 minute delivery socks" generateGEOContent( keywords: readMem0('research.keywords'), brand: readMem0('brand'), competitors: readMem0('research.competitors') ) → { seo_plan, geo_faqs, content_calendar }
Clarification questions asked upfront

    Is the brand targeting Hindi-language queries too?
    Do you want a blog strategy or just on-page SEO?

Agent 08 Required
Launch Report Agent
Synthesis · brand bible · 90-day roadmap

Reads everything from Mem0. Compiles a single Brand Launch Bible: brand overview, visual identity, domain recommendation, GTM plan, Shopify setup guide, ad strategy, SEO plan, and a 90-day milestone calendar. Outputs as a formatted markdown document and a structured JSON the Orchestrator can display.
// Final output contract interface LaunchBible { brand: BrandSummary; visual: { logo_urls: string[], palette: string[] }; domain: { recommended: string, alternatives: string[] }; gtm: IndiaGTMPlan; shopify_files: ShopifyPackage; ads: AdsStrategy; seo_geo: SEOGEOPlan; roadmap_90d: Milestone[]; }
02 Shared Memory
Every agent reads
what came before.

Use Mem0 (or a compatible in-memory equivalent with the same read/write API) as the shared knowledge store. Every agent reads from it before acting. Every agent writes back structured output after completing. This is what makes each step smarter than the last.
// Mem0 schema contract — all agents must conform to this shape // typescript: mastra/memory/schema.ts interface OpenClawMemory { // Written by: Orchestrator (on start) idea: { raw: string; // original user input brand_name_candidates: string[]; category: string; // "D2C fashion" | "FMCG" | "quick-commerce" }; // Written by: Research Agent research: { competitors: Competitor[]; market_size_inr: string; whitespace: string; keywords: { primary: string[], secondary: string[] }; india_insight: string; }; // Written by: Visual Agent visual: { brand_name: string; // confirmed name logo_concepts: LogoConcept[]; // 3 concepts with URLs chosen_concept: number; // user-selected: 0|1|2 palette: string[]; // hex codes font_pairing: string; mood: string; // "playful" | "premium" | "bold" }; // Written by: Domain Agent domains: { recommended: string; top5: DomainResult[]; }; // Written by: India GTM Agent gtm: IndiaGTMPlan; // Written by: Shopify Agent shopify: ShopifyPackage; // Written by: Ads Agent ads: AdsStrategy; // Written by: SEO/GEO Agent seo: SEOGEOPlan; // Append-only log — all agents write here audit_log: { agent: string; action: string; timestamp: string; keys_written: string[]; }[]; }
Memory Compounding Requirement

The Shopify Agent must demonstrate memory compounding. Its generated theme colors must come directly from mem0.visual.palette. Its product descriptions must use keywords from mem0.research.keywords. The Ads Agent must not ask "what's the brand name?" — it reads from Mem0. This is the assignment's central test: does memory actually flow, or is each agent starting from scratch?

06 Evaluation
How we score
your work.

Architecture decisions matter more than completions. A system where 5 agents work beautifully and memory compounds correctly scores higher than 8 agents where each one starts from scratch.
Criterion 	What we look for 	Weight
Memory Compounding 	Does each agent provably read from Mem0 before acting? Does the Shopify agent use the exact palette from the Visual agent? Does the Ads agent never ask for info that's already in memory? 	Core
Agent Boundaries 	Are agents well-scoped? Does each agent do exactly one thing? Is there no logic bleed between agents (e.g., Research agent writing ad copy)? 	Core
Workflow DAG 	Is the agent execution order correct? Do parallel steps actually run in parallel? Does the Orchestrator handle a mid-run failure gracefully? 	Core
Clarification Design 	Do agents ask questions at the right time (upfront, not mid-generation)? Are questions batched to avoid back-and-forth spam? Is the user experience conversational? 	Core
Nanobanana 2 Usage 	Is the Visual Agent correctly integrated with Nanobanana 2? Does it pass brand context from Mem0 into the prompt? Does it generate 3 distinct concepts, not just 3 variations? 	Core
Tool Design 	Every tool has a typed Zod schema. Input/output contracts are clear. Errors surface and are handled. No tool does two unrelated things. 	Core
Shopify Output Quality 	Are the generated files actually usable? Does the theme JSON use real Dawn schema keys? Are product descriptions written in brand voice derived from Mem0? 	Important
GEO Strategy 	Does the SEO agent actually differentiate between classic SEO and generative engine optimisation? Are GEO pages structured to be cited by AI search, not just ranked by Google? 	Important
Parallel Execution 	Do Research + Domain run concurrently? Mastra Workflow DAG shows correct parallelisation. Measurable time saving vs sequential. 	Bonus
Human-in-the-loop 	Can the user interrupt, correct, and resume? E.g., user says "I don't like Concept B, regenerate" — does Visual Agent re-run without restarting the whole workflow? 	Bonus
Multi-brand Support 	Can the same system launch a second brand without state bleed? Is the Mem0 namespace isolated per run? 	Bonus
